## 1. 違法コンテンツ対応の体系

### 1.1 対応フローの全体像

```
通報受理
  ↓
【優先度判定】
  ├─ Critical（児童虐待・違法薬物等）
  │   → 即削除 + 警察通報
  │
  ├─ High（詐欺・強盗等）
  │   → 4時間以内調査 + 削除 + ユーザー対応
  │
  ├─ Medium（継続的ハラスメス等）
  │   → 24時間以内調査 + 警告 or 削除
  │
  └─ Low（軽微違反）
      → 72時間以内 判定 + 対応

対応後
  ↓
【記録・報告】
  └─ 対応ログ保存（2年）+ 法執行機関対応（必要時）
```

### 1.2 判定基準の詳細

| 優先度 | コンテンツ種別 | 法的根拠 | SLA | 対応内容 | 警察等連携 |
|--------|----------|--------|-----|--------|---------|
| **Critical** | 児童虐待・CSAM | 児童福祉法・刑法 235条等 | 1時間 | 即削除 + 警察通報 | **必須** |
| **Critical** | 違法薬物販売 | 麻薬取締法 | 1時間 | 即削除 + 警察通報 | **必須** |
| **Critical** | テロ組織公式情報 | テロ対策法 | 1時間 | 即削除 + 情報確認 | **必須** |
| **High** | 詐欺・ねずみ講 | 刑法 246条等 | 4時間 | 調査 → 削除 | 推奨 |
| **High** | 脅迫・強要 | 刑法 222条等 | 4時間 | 調査 → 削除 + 被害者支援 | 推奨 |
| **High** | 著作権侵害（DMCA相当） | 著作権法 | 4時間 | 権利者確認 → 削除 | 不要 |
| **Medium** | 継続的ハラスメス（doxxing） | 刑法 162条等 | 24時間 | 警告 → 必要時削除 | 協力 |
| **Medium** | ポルノグラフィ（AI等） | 有害コンテンツ法 | 24時間 | 削除 + 注視 | 協力 |
| **Low** | 禁止行為の軽微違反 | TOS 4条 | 72時間 | 警告 or 削除 | なし |
| **Low** | スパム投稿 | TOS 4.3条 | 72時間 | 注視 → レート制限 | なし |

## 2. 高リスク領域のエスカレーションルール

### 2.1 児童虐待コンテンツ (CSAM)

**検出時の即座対応:**
```
1. 投稿即削除（ユーザーへの事前警告なし）
2. ユーザープロフィール確認
3. 警察児童虐待対応センター（NCMEC 相当）へ報告
   - URL捕捉・ハッシュ化
   - ユーザーIP・タイムスタンプ・メタデータ
4. 当該ユーザーに対して無条件アカウント停止
5. 法執行機関への自発的開示（令状不要）
6. 社内チーム全体への周知・再教育
```

**対応記録:**
- 削除投稿のメタデータ（hash値等）を暗号化保存
- 報告機関・報告日時・対応記録を 永続保持
- コンプライアンス委員会における定期報告

### 2.2 違法薬物関連コンテンツ

**判定基準:**
- 薬物の**使用方法・入手先・効能** を詳細に記述
- 薬物の配信・販売を勧誘
- **文学的言及**（小説内での言及等）は慎重に判定

**対応フロー:**
1. **削除確定の場合:**
   - 投稿即削除
   - ユーザーに警告メッセージ送信（薬物法違反の可能性を指摘）
   - 警察 or 厚労省への報告（配信・販売の場合は必須）

2. **文学的言及の場合（判断困難）:**
   - 法務層への相談（未実装：Phase 2）
   - 保留中の注視（コンテンツ削除しない）
   - ユーザーへの質問（「創作ですか？」）

### 2.3 詐欺・ねずみ講関連

**検出シグナル:**
- 「確実に儲かる」「簡単に稼げる」という金銭勧誘
- マルチレベルマーケティング（MLM）の勧誘
- 投資詐欺の勧誘

**対応:**
1. 投稿の事実確認
   - リンク先・連絡先の確認
   - 既報の詐欺グループとの照合
   
2. ユーザー確認
   - 初回投稿か常習者か
   - 他ユーザーからの通報歴
   
3. 対応実行
   - **初回（低信度）**: ユーザーへの注意メッセージ
   - **複数報告（高信度）**: 投稿削除 + 警告 + 必要時停止
   - **明白な詐欺**: 投稿削除 + 警察通報

### 2.4 脅迫・強要コンテンツ

**検出シグナル:**
- 特定個人への直接的脅迫（「〇〇しないと〜する」）
- 個人情報暴露の脅迫（恐喝目的）
- 自殺強要等

**対応:**
1. 被害者保護が最優先
   - 被害者へのコンプライアンスチームからの直接連絡
   - セキュリティ対応（ブロック・隔離等）
   
2. 投稿削除
   - すべての脅迫テキスト削除
   - スクリーンショットを証拠保存
   
3. 警察通報
   - 脅迫罪・強要罪は刑事犯
   - 被害者の同意なく通報可

## 3. 著作権侵害対応

### 3.1 DMCA 型通報受け取り

本サービスは 著作権保有者からの著作権侵害通報に対応します。

**通報フォーマット:**
```
件名: 著作権侵害通報 - 投稿ID [ID]

本文:
- 著作権所有者の氏名・連絡先
- 損害を受けた著作物の説明
- 侵害投稿の URL
- 侵害著作物との具体的な同一性
- 法的責任に対する誠実な声明
```

**対応:**
1. **受け取り**: 通報受け取り確認をメール送信
2. **調査**: 
   - 投稿内容の確認
   - 引用・変形の程度を判定（フェアユース判断）
3. **削除決定**:
   - 明白な侵害: 投稿削除 + ユーザー警告
   - グレーゾーン: 法務判定待機（Phase 2）
4. **ユーザー権利**:
   - ユーザーがカウンター通報可能（Notification of Claimed Infringement）
   - 両論を記録・保存

### 3.2 反復侵害者への対応

- **1回目**: 警告
- **2-3回目**: 投稿削除 + 厳重警告
- **4回以上**: 無期停止候補

## 4. 組織的対応体制

### 4.1 コンプライアンス委員会（構成予定）

| 役割 | 担当 | 責任 |
|------|------|------|
| **Head** | CEO / CTO | 最終判断責任 |
| **法務（予定）** | 外部弁護士 | 法的解釈・記録 |
| **運用** | モデレータ | 日次判定・削除実行 |
| **セキュリティ** | セキュリティ担当 | 本体システム保護・ログ管理 |
| **ユーザーサポート** | サポート担当 | ユーザーへの説明・異議受付 |

### 4.2 判定会議

- **週 1回（最小）**: 新規通報
- **臨機応変**: Critical 案件は即座に対応
- **月 1回**: 対応記録レビュー・ポリシー改善

### 4.3 異議申し立て手続

**ユーザーが処分に異議がある場合:**

1. **申請方法:**
   - メール: support@fiction-sns.example
   - 件名: 「異議申し立て: 投稿ID [ID]」
   - 本文: 具体的な異議理由

2. **審査:**
   - 受け取り確認を 1営業日以内に返送
   - 独立した者による再審査（30日以内）
   - 理由付き決定をメール通知

3. **結果:**
   - 異議認容: 処分取消 or 軽減
   - 異議棄却: 既存処分維持

## 5. 記録・監査・透明化

### 5.1 コンプライアンスログの保持

全通報・対応は以下を記録：
```
- 通報日時・通報者・対象（投稿 or ユーザー）
- 理由カテゴリ・詳細説明
- 優先度判定・SLA
- 調査内容・判定日時
- 対応内容（削除？警告？停止？）
- 決定者名・対応完了日時
- （法執行機関対応の場合）機関名・対応内容・日時
```

**保存期間:**
- 削除・警告対応: 2年
- 法執行機関対応: 永続保持（監査対応）

### 5.2 内部監査

- **月1回**: ランダムサンプリング監査（判定妥当性確認）
- **四半期**:全件レビュー（傾向分析・ポリシー改善）
- **年1回**: 外部監査（Phase 2）

### 5.3 透明性報告（Phase 2）

年1回、以下を公開予定：
- 通報受け取り件数（カテゴリ別）
- 処分実行件数
- 異議申し立て件数・認容率
- 法執行機関対応件数

## 6. ユーザーへの情報提供

### 6.1 処分通知

投稿削除・アカウント停止時：
```
メール例：

件名: fiction-sns - 投稿削除のお知らせ

本文:
投稿ID: [ID]
削除日時: [日時]
理由: 利用規約 4.2条（ハラスメント）違反
具体内容: 他ユーザーへの継続的な誹謗中傷

異議がある場合:
support@fiction-sns.example へ 30日以内にご連絡ください。
```

### 6.2 ユーザー教育

- **ヘルプセンター**: 違反例・対応フロー を掲載
- **新規ユーザー**: 登録時に TOS・禁止行為を表示
- **定期通信**: ニュースレターで重要な判定例を共有

## 7. 法執行機関への対応

### 7.1 令状・召喚状への対応

**受け取り時の手順:**

1. **確認**: 令状が有効な文書か確認
2. **記録**: すべての開示要求を記録
3. **対応**: 30日以内（又は令状の指定期限内）に開示
4. **ユーザー通知**: 開示したことをユーザーに通知（例外：捜査妨害懸念時は通知延期可）

**開示範囲:**
- ユーザーID・メールアドレス
- IP アドレス・ログイン日時
- 投稿内容・タイムスタンプ
- その他令状で指定された範囲

### 7.2 自発的開示（児童虐待等）

以下の場合は令状なく自発的に法執行機関に開示：
- 児童虐待コンテンツ
- 差し迫った危険（爆漸等の予告）
- テロ・ハイジャック等
