# スマホ実機E2E検証シナリオ（M4-14）

本文書は、Milestone 4 の公開可否判定に向けて、スマートフォンからのユーザーフロー全体を検証するためのシナリオと実行チェックリストをまとめたものです。

## 1. 検証環境

### 1-1. 開発者検証（ローカル）
- ブラウザ：Chrome / Firefox（DevTools のモバイルエミュレーション）
- ビューポート：375 × 812（iPhone SE / 12 mini相当）
- ネットワーク：通常 / 低速（DevTools Throttle: Fast 3G）

### 1-2. 実機検証（将来）
- 実際のスマートフォン（iOS / Android）
- 公開URL へのアクセス
- 4G / 5G / WiFi ネットワーク

## 2. E2Eシナリオ（主要導線）

### 2-1. シナリオ全体フロー
```
1. 新規ユーザー登録（signup）
   ↓
2. ログイン
   ↓
3. World 作成
   ↓
4. Character 作成・持ち込み
   ↓
5. 投稿（タイムライン投稿）
   ↓
6. タイムライン閲覧・ページング
   ↓
7. 他ユーザーの公開プロフィール閲覧
   ↓
8. フォロー・フォロー解除
   ↓
9. フォロー中・フォロワー一覧確認
   ↓
10. ログアウト
```

### 2-2. 詳細ステップ

#### ステップ 1: 新規ユーザー登録

| 項目 | チェック項目 |
|------|-----------|
| **URL** | /auth/signup/ |
| **表示** | フォーム（username / email / password / password_confirm / handle）が画面内に収まるか |
| **操作** | 各フィールドにテストデータ入力（username: testuser, email: test@example.com, handle: testuser） |
| **送信** | 登録ボタンを押下 |
| **確認** | ✅ ログイン画面へ遷移 / ✅ エラーメッセージが明確 |
| **モバイル** | フォーム送信後、キーボード が自動的に閉じるか |

#### ステップ 2: ログイン

| 項目 | チェック項目 |
|------|-----------|
| **URL** | /auth/login/ |
| **表示** | ログインフォーム（email or username / password）が画面内に収まるか |
| **操作** | ユーザー認証情報を入力・送信 |
| **確認** | ✅ ダッシュボード（/users/dashboard/）へ遷移 |
| **モバイル** | スマホキーボード操作は快適か / 戻るボタンで前のページに戻れるか |

#### ステップ 3: World 作成

| 項目 | チェック項目 |
|------|-----------|
| **URL** | /worlds/create/ |
| **表示** | フォーム（title / description / visibility）が見やすいか |
| **操作** | title: "Test World" / description: "Test" / visibility: public |
| **確認** | ✅ World 一覧に新規作成 World が表示される |
| **モバイル** | visibility 選択肢（ラジオボタン）がタップしやすいか |

#### ステップ 4: Character 作成・持ち込み

| 項目 | チェック項目 |
|------|-----------|
| **URL** | /worlds/<world_id>/character/create/ |
| **表示** | フォーム（name / profile / personality）のレイアウト |
| **操作** | name: "Hero" / profile: "A brave hero" / personality: "Brave" で作成 |
| **確認** | ✅ Character 一覧に追加される / ✅ World タイムラインで Character が選択可能 |
| **モバイル** | 複数行テキストフィールド（profile）の入力体験 |

#### ステップ 5: 投稿（タイムライン投稿）

| 項目 | チェック項目 |
|------|-----------|
| **URL** | /worlds/<world_id>/post/create/ |
| **表示** | Character 選択フォーム・テキスト入力フィールドが見やすいか |
| **操作** | Character 選択 + "Hello from mobile!" を投稿 |
| **確認** | ✅ タイムラインに投稿が即座に表示される |
| **モバイル** | キーボード出現時にフォーム全体が見えるか / スクロール可能か |

#### ステップ 6: タイムライン閲覧・ページング

| 項目 | チェック項目 |
|------|-----------|
| **URL** | /worlds/<world_id>/timeline/ |
| **表示** | タイムラインアイテム（投稿）が 1カラム・読みやすいか |
| **操作** | 投稿を 5個以上作成して「次の投稿を読み込む」ボタンをタップ |
| **確認** | ✅ 次ページが読み込まれる（重複なし） / ✅ 再試行ボタンが表示される（失敗時） |
| **モバイル** | スクロール・タップの反応速度は快適か / ボタンサイズはタッチしやすいか（最小48×48px推奨） |
| **低速環境** | Fast 3G でのロード時間は許容範囲（p95 < 2秒）か |

#### ステップ 7: 他ユーザーの公開プロフィール閲覧

| 項目 | チェック項目 |
|------|-----------|
| **URL** | /users/<handle>/profile/ |
| **表示** | プロフィール情報（Display Name / @handle / bio / フォロー情報）が画面内に収まるか |
| **操作** | 別ユーザーのプロフィールを確認 |
| **確認** | ✅ フォローボタンが表示される / ✅ プロフィール画像スペース（将来実装） |
| **モバイル** | プロフィール情報の階層構造が理解しやすいか |

#### ステップ 8: フォロー・フォロー解除

| 項目 | チェック項目 |
|------|-----------|
| **URL** | /users/<handle>/profile/ |
| **操作** | フォローボタン をタップ |
| **確認** | ✅ ボタンが「フォロー中」に変わる / ✅ ページリロード後も状態が保持される |
| **操作** | フォロー中ボタン をタップ（解除） |
| **確認** | ✅ ボタンが「フォロー」に戻る |
| **モバイル** | ボタンの状態変化がユーザー周知できるか（色・テキスト変更） |

#### ステップ 9: フォロー中・フォロワー一覧確認

| 項目 | チェック項目 |
|------|-----------|
| **URL** | /users/dashboard/ → フォロー中・フォロワー一覧リンク |
| **表示** | ユーザー一覧（@handle / Display Name / bio）が読みやすいか |
| **操作** | 一覧から他ユーザーをタップ → プロフィール遷移 |
| **確認** | ✅ 一覧表示・タップ遷移が正常 |
| **モバイル** | リストアイテムのタップ領域は十分か（44×44px以上推奨） |

#### ステップ 10: ログアウト

| 項目 | チェック項目 |
|------|-----------|
| **URL** | ヘッダーのログアウトボタン |
| **操作** | ログアウトボタンをタップ |
| **確認** | ✅ トップページへ遷移 / ✅ ログイン状態がクリア |
| **モバイル** | ナビゲーション・ボタンの可視性・タップしやすさ |

## 5. 判定基準

### Go判定（全クリア）
- ✅ 10ステップシナリオ完走
- ✅ エラー発生なし
- ✅ UX・パフォーマンス許容範囲

### Conditional Go（小修正で対応可）
- ❌ 軽微なUI崩れ（レイアウト・色調のみ）
- ❌ 予期した動作に数秒の遅延
- 📋: 修正Issue作成 → #65で再判定

### No-Go（ブロッカー級）
- ❌ フロー中断エラー（ログイン不可、投稿失敗等）
- ❌ データ喪失（投稿後に消える等）
- ❌ セキュリティ問題（認証回避等）
